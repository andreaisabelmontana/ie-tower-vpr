A runnable image-retrieval system for place recognition around the IE Tower. A query is turned into a classical global descriptor — ORB bag-of-visual-words plus colour and gradient histograms — matched against a gallery with exact cosine nearest-neighbour search, and its location is predicted by voting over the top matches.

01 — Pipeline

From a query view to a location.

One classical descriptor per image, one retrieval index, a top-K vote. No training, no GPU — just hand-crafted features and exact nearest-neighbour search.

DESCRIBE

ORB bag-of-visual-words

ORB keypoints quantised against a KMeans visual-word vocabulary learned from the gallery, plus HSV colour and gradient-orientation histograms — concatenated and L2-normalized.

SEARCH

Exact cosine NN index

One gallery descriptor matrix; a single matrix–vector product gives exact top-K. Stands in for the FAISS index used in the original group project.

03 — Honest scope

What's here, and what isn't.

This repo implements and runs the classical retrieval track for real. It does not fake the deep tracks.

The original BCSAI Computer Vision group project fused three tracks — classical (local features), DINOv2 ViT embeddings, and a supervised CNN — into a single FAISS index. This repository ships only the classical track: the DINOv2/CNN backbones and the FAISS index need a GPU / PyTorch, which aren't available here. So the index uses exact cosine nearest-neighbour search (numpy/scikit-learn) over the classical descriptors instead, and only metrics from real runs are reported.
